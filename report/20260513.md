## go-yaml

- [ ] [PR #348 Fix !!merge tag regression for yq #2677](https://github.com/yaml/go-yaml/pull/348)
  Upstream fix for the round-trip regression introduced by commit cb0eff8 where the implicit `!!merge` tag was no longer being elided on encode, producing output like `!!merge <<: *x` instead of `<<: *x`.
  Splits `binaryTag`/`mergeTag` handling so the implicit merge tag is stripped while explicit `TaggedStyle` preservation is kept.
  Directly unblocks the yq regression in mikefarah/yq#2677.
  (updated: 2026-05-13T13:06:37Z, appearance #1)

- [ ] [PR #347 [v3 backport] Replace gopkg.in/check.v1 with the standard testing library](https://github.com/yaml/go-yaml/pull/347)
  Backport of PR #52 onto v3, swapping the `gopkg.in/check.v1` dependency for stdlib `testing` (+669/-336 across 7 files, 4 comments of active discussion).
  Trims an external dep from the still-widely-used v3 line.
  (updated: 2026-05-13T13:54:46Z, appearance #1)

- [ ] [#344 v3: corrupted godoc after gofmt in parserc.go, yamlh.go](https://github.com/yaml/go-yaml/issues/344)
  New v3 regression: commit 5f3d8c3 ran `gofmt` and mangled godoc layout in twelve flagged ranges across `parserc.go` and `yamlh.go`.
