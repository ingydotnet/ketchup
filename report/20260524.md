## go-yaml

- [ ] [#354 Emoji/unicode is incorrectly escaped](https://github.com/yaml/go-yaml/issues/354)
  New comments landed on 2026-05-23, extending the discussion from the dasel maintainer's initial report. The go-yaml encoder escapes emoji and non-ASCII Unicode to `\uXXXX` sequences while JSON/TOML encoders leave them as-is — a correctness gap affecting any tool using go-yaml to emit YAML containing non-ASCII content. An unmerged upstream fix has been referenced but has not landed.
