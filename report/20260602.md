## yamlscript

- [ ] [#267 macOS arm64: installer Makefile uses cp libys*.so* (Linux glob) — fails on macOS](https://github.com/yaml/yamlscript/issues/267)
  New comments today advanced the timestamp from 2026-05-07 to 2026-06-02. The installer Makefile glob `libys*.so*` only matches Linux `.so` libraries, silently failing on macOS arm64 where libraries use `.dylib` extensions. A fix using `$(LIBYS)*` to capture the correct platform-specific filename has been proposed in-thread but the issue remains open.
  (updated: 2026-06-02T07:22:46Z, appearance #4)

## yq

- [ ] [#439 folded multiline scalars should stay in their original format](https://github.com/mikefarah/yq/issues/439)
  New activity today: folded (`>`) and plain multiline scalars lose their original style after yq operations because go-yaml's round-trip handling does not fully preserve scalar style in the Node representation. The reporter explicitly asks whether the fix belongs in go-yaml, placing this in the same design space as the ongoing v4 API discussion in thread #253.
  (updated: 2026-06-02T07:31:44Z, appearance #1)

## pyyaml

- [ ] [#860 Scientific notation is not properly quoted](https://github.com/yaml/pyyaml/issues/860)
  New activity today: strings like `1E6` are emitted unquoted by PyYAML's dumper, causing other parsers (notably go-yaml) to interpret them as floating-point numbers rather than strings. This is a cross-implementation compatibility gap affecting any pipeline that exchanges YAML between PyYAML and go-yaml or similar implementations.
  (updated: 2026-06-01T21:08:35Z, appearance #1)

- [ ] [#933 safe_load raises raw KeyError/ValueError/IndexError instead of ConstructorError for !!bool/!!int/!!float](https://github.com/yaml/pyyaml/issues/933)
  Re-surfaced: safe_load surfaces raw Python exceptions when parsing malformed explicit type tags (e.g. `!!bool not-a-bool`), breaking any code that catches `yaml.YAMLError` for graceful error handling. Fix PR #934 was opened the same day and remains unmerged with no new review activity since.
  (updated: 2026-05-24T13:18:11Z, appearance #2)

## go-yaml

- [ ] [#354 Emoji/unicode is incorrectly escaped](https://github.com/yaml/go-yaml/issues/354)
  Re-surfaced: the go-yaml encoder escapes emoji and non-BMP Unicode to `\uXXXX` sequences instead of emitting them directly, diverging from JSON and TOML encoders and breaking pipelines that produce YAML with non-ASCII content. Fix PR #357 (opened 2026-05-25) has seen no new commits and remains unmerged.
