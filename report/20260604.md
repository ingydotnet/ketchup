Fix PR #359 was opened the same day and is under review, but the issue remains open and no stable fix has shipped.

## yq

- [ ] [PR #2730 fix: preserve correct parent references in explode merge anchor reconstruction](https://github.com/mikefarah/yq/pull/2730)
  New PR by vomba fixing a bug in yq's anchor/merge reconstruction that surfaces when using `--yaml-fix-merge-anchor-to-spec` (yq's workaround for go-yaml's non-spec-compliant !!merge handling). Root cause: `item.Copy()` preserved stale parent pointers from alias targets in `fixedReconstructAliasedMap`, and `AddChild` was used instead of `AddKeyValueChild` in `reconstructAliasedMap`, both corrupting traversal paths during explode + merge operations. Closes issue #2434.
  (updated: 2026-06-03T12:46:37Z, appearance #1)

- [ ] [PR #2705 Fix for #2677](https://github.com/mikefarah/yq/pull/2705)

## pyyaml

- [ ] [#931 No win32 wheel for Python3.14](https://github.com/yaml/pyyaml/issues/931)
  Re-surfaced: the 6.0.3 release deliberately dropped win32 wheels for Python 3.14, breaking Azure CLI's embedded-Python build pipeline. Python 3.14 still ships a 32-bit Windows installer, making the omission non-trivial for downstream consumers. The companion fix PR #932 was closed without merging on 2026-05-26, leaving the issue with no clear resolution path.
  (updated: 2026-05-26T16:59:42Z, appearance #4)

- [ ] [PR #934 fix: wrap raw exceptions in construct_yaml_bool/int/float as ConstructorError](https://github.com/yaml/pyyaml/pull/934)
  Re-surfaced: `yaml.safe_load` raises raw `KeyError`, `ValueError`, or `IndexError` on malformed explicit type tags (e.g. `!!bool maybe`, `!!int abc`), bypassing any `except yaml.YAMLError` guard. This PR wraps the affected constructors to raise `ConstructorError` consistently, adds 17 parametrised tests, and closes issue #933. Open and awaiting review since late May.
  (updated: 2026-05-25T12:40:05Z, appearance #2)
