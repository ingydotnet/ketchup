## go-yaml

- [ ] [#354 Emoji/unicode is incorrectly escaped](https://github.com/yaml/go-yaml/issues/354)
  Re-surfaced: the go-yaml encoder escapes emoji and supplementary-plane Unicode (code points above U+FFFF) to `\uXXXX` sequences rather than emitting them directly, diverging from JSON and TOML encoders and breaking round-trip fidelity for content with international text or emoji. Fix PR #357 was opened on 2026-05-25 and was last seen unmerged in the 2026-06-04 report; the issue is a persistent output-quality gap for any application producing YAML with non-ASCII content.

## pyyaml

- [ ] [#860 Scientific notation is not properly quoted](https://github.com/yaml/pyyaml/issues/860)
  Re-surfaced: PyYAML's dumper emits strings like `1E6` unquoted, causing other parsers (notably go-yaml) to interpret them as floating-point numbers rather than strings — a cross-implementation compatibility gap affecting any pipeline that exchanges YAML between PyYAML and go-yaml. No fix or linked PR has appeared since the issue was filed.
  (updated: 2026-06-01T21:08:35Z, appearance #2)

## yamlscript

- [ ] [#267 macOS arm64: installer Makefile uses cp libys*.so* (Linux glob) — fails on macOS](https://github.com/yaml/yamlscript/issues/267)
  Re-surfaced: the YAMLScript installer Makefile uses the glob `libys*.so*`, which only matches Linux `.so` dynamic libraries and silently fails on macOS arm64 where libraries use `.dylib` extensions. New in-thread comments on 2026-06-02 proposed a fix using `$(LIBYS)*` to match platform-specific filenames, but the issue remains open and unresolved, blocking clean installs on Apple Silicon Macs.
  (updated: 2026-06-02T07:22:46Z, appearance #5)
